## 1. 系统要求

### 硬件与操作系统

| 项目 | 要求 |
|------|------|
| 操作系统 | **Windows 10/11**（当前仅适配 Windows） |
| 剪映版本 | **剪映专业版 5.9 及以下**（6+ 版本加密了草稿文件，详见兼容性说明） |
| Python | 3.8+ |
| 网络 | TTS 配音需要联网（调用微软 edge-tts API） |

### 软件依赖

| 依赖包 | 用途 | 安装方式 |
|--------|------|----------|
| `edge-tts` | TTS 配音生成 | `pip install edge-tts` |
| `opencv-python-headless` | 关键帧提取、视频分析 | `pip install opencv-python-headless` |
| `mutagen` | MP3 精确时长读取 | `pip install mutagen` |
| `ffmpeg / ffprobe` | 视频时长/分辨率分析（可选，有 cv2 回退） | 系统级安装 |
| `numpy` | cv2 的依赖（自动随 opencv 安装） | 自动安装 |
| `tabulate` | 格式化输出 | `pip install tabulate` |

### 前置条件

1. **剪映已安装**：在 Windows 上安装剪映专业版（JianyingPro）
2. **至少创建过一次草稿**：首次使用剪映时，需要手动创建一个草稿项目，以便 Skill 能找到剪映的草稿目录
3. **Python 环境可用**：需要 Python 3.8 及以上版本

## 2. 安装步骤

### 方式 A：Skill 安装（推荐）

将整个 `jianying-video-editor` 文件夹放到 Skill 安装目录下即可。

### 方式 B：手动下载

将 Skill 仓库克隆或下载到 Skill 安装目录。

### 配置 Python 环境

**强烈建议**创建独立虚拟环境，避免污染全局 Python：

```bash
# 1. 创建虚拟环境（请替换 <你的虚拟环境路径>）
python -m venv <你的虚拟环境路径>

# 2. 安装依赖
<你的虚拟环境路径>\Scripts\pip.exe install \
  edge-tts opencv-python-headless mutagen tabulate

# 3. 验证安装
<你的虚拟环境路径>\Scripts\python.exe -c "
import edge_tts, cv2, mutagen
print('所有依赖安装成功 ✓')
"
```

> 💡 你可以自行选择虚拟环境的存放位置，例如项目目录下或专用目录中。

### 修改脚本中的路径

安装完成后，需要修改 `jianying_edit.py` 中以下路径变量，使其指向你的实际环境：

```python
# scripts/jianying_edit.py 中需要修改的路径变量：
PYTHON = "你的虚拟环境中 python.exe 的路径"
SKILLS_DIR = "Skill 安装目录下的 scripts 子目录路径"
ASSETS_DIR = "Skill 安装目录下的 assets 子目录路径"
```

### 验证环境

```bash
<你的虚拟环境路径>\Scripts\python.exe \
  <Skill安装目录>\scripts\check_env.py
```

输出应显示：
```
=== jianying-video-editor 环境检查 ===

[OK] Python: 3.x.x
[OK] ffprobe: ffprobe version ...（或提示 cv2 回退）
[OK] 草稿目录存在
[OK] 目录可写: True

所有检查通过，可以开始使用
```

## 3. 剪映版本兼容性

### ⚠️ 重要警告

**剪映 6+ 版本对 `draft_content.json` 进行了加密处理**，这意味着：

- **读取已有草稿**：6+ 版本加密了草稿文件，无法直接读取/修改
- **注入新生成的草稿**：理论上可行（剪映打开新草稿时会读取明文 JSON，然后自己加密保存），但**未经充分验证**

| 剪映版本 | 草稿生成 | 草稿读取 | 批量导出 |
|---------|---------|---------|---------|
| **5.x（含 5.9）** | ✅ 完全兼容 | ✅ 完全兼容 | ✅ 完全兼容 |
| **6.x** | ⚠️ 可能兼容 | ❌ 加密不可读 | ✅ 兼容（Windows） |
| **7.x** | ⚠️ 可能兼容 | ❌ 加密不可读 | ❌ 隐藏了控件 |
| **Mac 版剪映** | ❌ 不兼容 | ❌ 不兼容 | ❌ 不兼容 |
| **CapCut（国际版）** | ❌ 不兼容 | ❌ 不兼容 | ❌ 不兼容 |

### 推荐做法

- 使用**剪映专业版 5.9 及以下版本**
- 不要升级到 6+ 版本，或保持两个版本并存
- Mac 用户：目前不支持，后续版本会适配

### 如何查看剪映版本

在剪映中点击「设置」→「关于剪映」即可查看当前版本号，例如 `5.9.0.11632`。

## 4. 快速开始

### 最简示例：视频拼接

```bash
PYTHON="你的虚拟环境中 python.exe 的路径"
SCRIPTS="Skill 安装目录下的 scripts 子目录路径"

$PYTHON $SCRIPTS/jianying_edit.py \
  --name "我的第一个视频" \
  --files video1.mp4 video2.mp4 video3.mp4
```

打开剪映 → 本地草稿 → 找到「我的第一个视频」 → 即可查看和导出。

### 加字幕 + 配音 + 转场

```bash
$PYTHON $SCRIPTS/jianying_edit.py \
  --name "旅行Vlog" \
  --files clip1.mp4 clip2.mp4 clip3.mp4 \
  --narration-json '[{"index":0,"text":"清晨的海边"}, {"index":1,"text":"漫步沙滩"}, {"index":2,"text":"日落余晖"}]' \
  --tts-voice zh-CN-YunxiNeural \
  --transitions dissolve \
  --subtitle-style outline_clean \
  --style vlog
```

## 5. 使用教程

### 模式一：智能旁白（推荐）

当你想让字幕/配音**跟画面内容相关**时，使用此模式。

**总流程**：

```
视频素材 → 提取关键帧 → 识别画面写旁白 → 生成草稿 → 注入剪映
```

#### Step 1 — 提取关键帧

```bash
$PYTHON $SCRIPTS/extract_frames.py \
  --files video1.mp4 video2.mp4 video3.mp4 \
  --output-dir ./keyframes \
  --frames-per-clip 3
```

- `--frames-per-clip`：每段视频提取多少帧（3-5 推荐，越多看得越清楚）
- 输出 JSON，包含每段视频的帧图片路径

#### Step 2 — 分析画面，写旁白

**这一步不是脚本完成的，需要人工完成**：

1. 读取上一步生成的帧图片
2. 分析每段视频的画面内容
3. 为每段写一句旁白文案（15-40 字，描述画面实际内容）
4. 生成 narration JSON：

```json
[
  {"index": 0, "text": "无人机在低空缓缓掠过展会大厅"},
  {"index": 1, "text": "大疆工业级无人机搭配禅思云台相机"},
  {"index": 2, "text": "美团配送无人机正在进行演示飞行"}
]
```

**旁白文案技巧**：
- 每段 1-2 句话，15-40 字
- 描述画面实际内容，不要泛泛而谈
- 根据风格调整语气（Vlog 活泼、纪录片沉稳）
- 可以有叙事性、抒情性，但必须跟画面对应

#### Step 3 — 一键生成草稿

```bash
# 方式 A：直接传 JSON 字符串
$PYTHON $SCRIPTS/jianying_edit.py \
  --name "无人机博览会" \
  --files video1.mp4 video2.mp4 video3.mp4 \
  --narration-json '[{"index":0,"text":"..."},{"index":1,"text":"..."}]' \
  --tts-voice zh-CN-YunxiNeural \
  --transitions dissolve \
  --subtitle-style yellow_accent

# 方式 B：传 JSON 文件（长文案推荐）
$PYTHON $SCRIPTS/jianying_edit.py \
  --name "无人机博览会" \
  --files video1.mp4 video2.mp4 video3.mp4 \
  --narration-file narration.json \
  --tts-voice zh-CN-YunxiNeural \
  --style documentary
```

脚本会自动：
1. 将旁白文案对齐到各段视频的时间轴
2. 调用 edge-tts 生成配音 MP3
3. 构建完整草稿（视频 + 字幕 + 配音 + 转场）
4. 注入剪映草稿目录

---

### 模式二：手动 SRT 字幕

如果你已有 SRT 字幕文件：

```bash
$PYTHON $SCRIPTS/jianying_edit.py \
  --name "带字幕的视频" \
  --files video1.mp4 video2.mp4 \
  --srt subtitle.srt \
  --tts \
  --tts-voice zh-CN-YunjianNeural \
  --subtitle-style cinema_bar
```

SRT 文件格式示例：

```srt
1
00:00:00,000 --> 00:00:05,000
第一段字幕文本

2
00:00:05,000 --> 00:00:10,000
第二段字幕文本
```

---

### 模式三：纯视频拼接（无字幕/配音）

最简单的模式——只是把多段视频拼在一起：

```bash
$PYTHON $SCRIPTS/jianying_edit.py \
  --name "简单拼接" \
  --files clip1.mp4 clip2.mp4 clip3.mp4 \
  --transitions dissolve
```

可选加上 BGM：

```bash
$PYTHON $SCRIPTS/jianying_edit.py \
  --name "简单拼接+BGM" \
  --files clip1.mp4 clip2.mp4 clip3.mp4 \
  --bgm background_music.mp3 \
  --bgm-volume 0.3
```

## 6. 完整参数说明

### 主脚本 `jianying_edit.py` 参数

| 参数 | 必填 | 类型 | 说明 |
|------|------|------|------|
| `--name` | ✅ | string | 草稿名称（在剪映中显示的名字） |
| `--files` | ✅ | string[] | 视频素材路径（多个，建议用绝对路径） |
| `--narration-json` | ❌ | string | 旁白 JSON 字符串（自动启用 TTS+字幕） |
| `--narration-file` | ❌ | string | 旁白 JSON 文件路径（同上） |
| `--bgm` | ❌ | string | BGM 音频文件路径 |
| `--bgm-volume` | ❌ | float | BGM 音量 0.0~1.0，默认 0.5 |
| `--srt` | ❌ | string | 字幕 SRT 文件路径（与 narration 二选一） |
| `--tts` | ❌ | flag | 启用 TTS 配音（配合 `--srt`；narration 自动启用） |
| `--tts-voice` | ❌ | string | TTS 语音 ID，默认 `zh-CN-YunxiNeural` |
| `--tts-rate` | ❌ | string | TTS 语速调整，如 `"+20%"` `"-10%"` |
| `--tts-volume` | ❌ | float | TTS 配音音量，默认 1.0 |
| `--transitions` | ❌ | string | 转场类型 ID，默认 `none`（详见转场表） |
| `--transition-duration` | ❌ | int | 转场时长（微秒），不填则用各转场默认值 |
| `--subtitle-style` | ❌ | string | 字幕样式 ID，默认 `default`（详见样式表） |
| `--style` | ❌ | string | 风格预设 ID，自动匹配转场/字幕/音色（详见风格表） |
| `--duration` | ❌ | int | 目标时长（秒），0 = 使用全部素材 |
| `--resolution` | ❌ | string | 输出分辨率，默认 `1920x1080` |
| `--fps` | ❌ | int | 帧率，默认 30 |
| `--cut-mode` | ❌ | string | 剪辑模式：`sequential`（顺序）/ `smart`（智能） |
| `--no-inject` | ❌ | flag | 不自动注入剪映，只保存到本地目录 |

### 关键帧提取 `extract_frames.py` 参数

| 参数 | 必填 | 说明 |
|------|------|------|
| `--files` | ✅ | 视频素材路径 |
| `--output-dir` | ❌ | 帧图片输出目录，默认 `./keyframes` |
| `--frames-per-clip` | ❌ | 每段提取帧数，默认 3 |
| `--format` | ❌ | 图片格式 jpg/png，默认 jpg |

### 注入剪映 `inject_draft.py` 参数

| 参数 | 必填 | 说明 |
|------|------|------|
| `--draft-dir` | ✅ | 草稿输出目录（build_draft.py 生成的） |
| `--name` | ✅ | 草稿名称（在剪映中显示） |

## 7. 配置文件详解

### 字幕样式 `assets/subtitle_styles.json`

10 种预设样式，每种包含：

| 字段 | 说明 |
|------|------|
| `name` | 样式名称 |
| `description` | 描述 |
| `text_color` | 文字颜色 [R, G, B]（0.0~1.0） |
| `font_size` | 字号（剪映内部单位） |
| `alignment` | 对齐方式：0=左, 1=居中, 2=右 |
| `position_y` | Y轴位置（归一化，负值=画面下方） |
| `background_style` | 背景样式：0=无, 1=有背景 |
| `stroke_width` | 描边宽度 |
| `stroke_color` | 描边颜色 |
| `has_shadow` | 是否有阴影 |
| `font_path` | 字体文件路径 |
| `font_title` | 字体名称 |

**自定义样式**：直接编辑此 JSON 文件，新增自己的样式 ID 即可。

### 转场预设 `assets/transition_presets.json`

12 种转场，每种包含：

| 字段 | 说明 |
|------|------|
| `name` | 转场名称 |
| `description` | 描述 |
| `duration_us` | 默认时长（微秒） |
| `effect_id` | 剪映特效 ID |
| `category` | 分类 |
| `verified` | 是否已验证可用（true=即插即用, false=需在剪映中下载） |

⚠️ `verified=false` 的转场需要先在剪映中下载对应资源包，否则转场不会显示。

### TTS 音色 `assets/tts_voices.json`

20+ 种语音，按分类整理：

| 分类 | 数量 | 说明 |
|------|------|------|
| 普通话·男声 | 7 | 云希/云健/云扬/云泽/云枫/云皓/云夏 |
| 普通话·女声 | 9 | 晓伊/晓辰/晓梦/晓墨/晓睿/晓双/晓萱/晓涵/晓甄 |
| 方言 | 2 | 晓北（东北）/ 晓妮（陕西） |
| 粤语 | 2 | 晓曼（女）/ 云龙（男） |

每种语音包含 `best_for`（适合内容类型）和 `style_to_voice`（风格→推荐音色映射），方便按内容类型快速选择。

## 8. 文件结构

```
jianying-video-editor/
├── SKILL.md                        ← Skill 描述文件
├── 安装与使用文档.md                ← 本文件
├── scripts/
│   ├── jianying_edit.py             ← ★ 一键入口脚本（推荐使用）
│   ├── analyze_media.py             ← 素材分析（ffprobe + cv2 回退）
│   ├── extract_frames.py            ← 关键帧提取（旁白模式用）
│   ├── build_draft.py               ← 构建 draft_content.json
│   ├── inject_draft.py              ← 写入剪映草稿目录
│   ├── tts_generate.py              ← TTS 配音生成
│   ├── check_env.py                 ← 环境检查脚本
│   └── jianying_drafts/             ← 草稿输出目录（自动创建）
├── references/
│   ├── draft_schema.md              ← 剪映草稿 JSON 结构文档
│   └── effect_catalog.md            ← 转场/特效 ID 目录
└── assets/
    ├── transition_presets.json       ← 转场预设配置
    ├── subtitle_styles.json          ← 字幕样式配置
    └── tts_voices.json               ← TTS 音色配置
```

### 各脚本职责

| 脚本 | 调用方式 | 说明 |
|------|---------|------|
| `jianying_edit.py` | 直接调用 | **一键入口**，串联所有步骤：分析→构建→TTS→注入 |
| `analyze_media.py` | 由入口脚本调用 | 分析视频时长/分辨率/音频 |
| `extract_frames.py` | 可独立调用 | 提取关键帧图片 |
| `build_draft.py` | 由入口脚本调用 | 构建草稿 JSON |
| `tts_generate.py` | 由入口脚本调用 | 生成 TTS 配音 MP3 |
| `inject_draft.py` | 可独立调用 | 将草稿注入剪映目录 |

## 9. 故障排查

### 常见问题

| 问题 | 原因 | 解决方案 |
|------|------|----------|
| **剪映看不到草稿** | 直接 cp 文件缺少 `draft_meta_info.json` | **必须用 `inject_draft.py` 注入**，不要直接复制文件 |
| **草稿打开报错** | 素材路径不存在 | 检查视频和 TTS 音频路径是否为有效绝对路径 |
| **字幕和配音不同步** | 时间基准不一致 | v4.4+ 已修复，确保使用最新版本 |
| **TTS 生成失败** | 网络不通或语音 ID 错误 | 检查网络连接；确认语音 ID 正确（见音色表） |
| **转场不显示** | 转场资源未下载 | 在剪映中手动下载对应转场资源包（`verified=false` 的转场） |
| **ffprobe 未找到** | ffmpeg 未安装 | 安装 ffmpeg，或 Skill 自动使用 cv2 回退 |
| **cv2 无法分析视频** | opencv 未安装 | `pip install opencv-python-headless` |
| **旁白与画面不匹配** | 提取帧数不够 | 增加 `--frames-per-clip 5`，或手动调整 narration JSON |

### 注入问题专项排查

如果草稿注入后剪映中看不到：

1. **确认用了 inject_draft.py**
   ```bash
   # 不要这样做 ❌：
   cp -r ./jianying_drafts/xxx "剪映草稿目录/xxx"

   # 正确做法 ✅：
   $PYTHON $SCRIPTS/inject_draft.py --draft-dir ./jianying_drafts/xxx --name "xxx"
   ```

2. **检查 root_meta_info.json**
   在剪映的草稿索引文件中确认是否有你的草稿条目，且条目中有 `draft_json_file` 字段（这是剪映定位草稿的关键）。

3. **重启剪映**
   注入后必须重启剪映才能看到新草稿。

4. **检查草稿目录完整性**
   ```
   草稿目录/应包含：
   ├── draft_content.json      ← 必须
   ├── draft_meta_info.json    ← 必须（inject 自动生成）
   └── textReading/            ← TTS 音频目录（如有配音）
       ├── tts_0000.mp3
       ├── tts_0001.mp3
       └── tts_meta.json
   ```

### 草稿目录位置

剪映草稿存储在剪映的 `User Data\Projects\com.lveditor.draft` 目录下。每个草稿是一个子目录，`root_meta_info.json` 是所有草稿的索引文件。
